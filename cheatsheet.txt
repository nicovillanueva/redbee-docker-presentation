Que es Docker
    En alto nivel, "una VM liviana"
        Tiene sus propios procesos
        Su propia red
        Puede correr procesos root internamente
    A bajo nivel, entornos de ejecucion aislados
        Los procesos son completamente aislados unos de otros
        Comparten kernel con el host
        No pueden emular hardware
    Docker no inventó nada
        Es un rejunte se elementos ya existentes en Unix
            namespaces(aislamiento de procesos, user ids, filesystem)
            cgroups (cpu, memoria, i/o, red)

Conceptos
    Layers
        "Capas" que forman una imagen
        Cada linea nueva en un dockerfile genera una layer: cada "RUN", o cada "ADD"
        Cada layer recibe un hash, y puede ser compartida entre builds
    Images
        Imagen completa, lista para correr
    Container
        Una "instancia" de una imagen
        La aplicación en sí
    DEMO: "docker pull busybox"

Dockerfile
    Archivo con sintaxis particular que describe una imagen a correr
    Conjunto de instrucciones que construyen una imagen
        FROM ubuntu:14.04
        RUN apt-get install python3
        ADD ./config.properties /root/app/config.properties
        CMD /bin/bash
    DEMO: build flask app

Comandos basicos
    build
    run
        -it
        --rm
        -v
        -p
    start
    ps
    exec
    inspect
    stop / kill
    rm / rmi
    DEMO: run flask app from image, start from container, ps -a, exec bash, inspect ip, stop, rm, rmi

Volumenes
    volumenes internos (-v)
    volumenes mappeados (-v x:y)
    DEMO: run busybox SIN volumen mappeado, touch, cerrar, volver; busybox CON volumen mappeado, touch, ver que está desde afuera

Otros proyectos
    Compose
        No describe una aplicacion particular, describe una arquitectura
        app1, app2, app3, db, message queue
        soporta scaling
        logging centralizado
    Machine
        Provisiona clientes de docker en distintos ambientes: localmente, datacenter interno, o cloud
            Localmente usa virtualbox y boot2docker (como lo hace Mac)
            Para datacenter soporta vmware, virtualbox
            Cloud soporta aws, digital ocean, rackspace, google compute engine, etc
        Instala, configura docker engine en los clientes
        Habilita el usar docker "remotamente" como si fuera "local"
    Swarm
        Orquesta la clusterizacion de hosts (funciona transparentemente con Machine)
